### Key Architecture Decisions

- `SessionStore` is the unified persistence interface — in-memory default, pluggable backend (SQLite, Redis) goes here when needed
- `PauseManager` uses `EventEmitter` internally — compatible with Node 22 `EventEmitter` (no `@types/node` dependency needed because package already has it)
- Crash detector uses heartbeat keys `session:heartbeat:<sessionId>` with `Date.now()` timestamps; threshold configurable per `detectStaleSessions()` call
- Validator produces `warnings[]` not `errors[]` for role alternation violations — these are non-fatal heuristics
- Reducer is pure function with no side effects — `forkSession` creates a new state object, doesn't mutate the original
