## NOTES & DECISIONS

### Key Architectural Decisions
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. **Package Granularity**: 37 packages vs planned 16
   - Decision: More granular boundaries preferred
   - Trade-off: Better tree-shaking, higher maintenance

2. **Core Consolidation Strategy**: 11 modules → single @agentsy/core
   - Decision: Unified stream processing in one package
   - Result: Simpler dependency graph, cleaner API surface

3. **Shim Package**: Created instead of bundle package
   - Decision: Preserve @agentsy/core API directly
   - Benefit: No breaking consumer imports

4. **Package Structure**: Subpath exports per consolidated module
   - Decision: "./processor", "./sse", etc. exports
   - Benefit: Selective imports still possible

5. **Build System**: Turborepo over single-package builds
   - Decision: Monorepo with dependency-aware caching
   - Benefit: Faster builds, better parallelization
