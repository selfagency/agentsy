2. **Core Package Depth: 11 modules consolidated**
   - Decision: processor, sse, structured, thinking, tool-calls, xml-filter, context, formatting, recovery, ui, normalizers
   - Trade-off: Larger core package, but unified stream processing
   - Benefit: Fewer packages, simpler dependency graph

3. **Normalizers Location: Moved INTO core**
   - Decision: Consolidate normalizers into @agentsy/core (not providers)
   - Reason: Product→Providers→Core→Product circular dependency issue
   - Resolution: normalizers now part of core package with subpath export

4. **Package Naming: Strategic renaming for clarity**
   - agentic-loop → runtime (clearer purpose)
   - token-economy → tokens (simpler, clearer naming)
   - agents → plugins (A2A protocol support, clearer branding)

5. **Build System: Turborepo with workspace resolution**
   - Decision: Monorepo with dependency-aware caching
   - Benefit: Faster builds, better parallelization
   - Status: Infrastructure operational, all packages building successfully

6. **Import Pattern: Subpath exports for granularity**
   - Decision: "./processor", "./sse", etc. exports from core
   - Benefit: Selective imports still possible despite consolidation
   - Status: Configured in package.json and tsup.config.ts

7. **Prompts Module: Token-optimized prompt management (Future Phase P13)**
   - Decision: Create @agentsy/prompts for prompt optimization and agentic patterns
   - Integration: Uses @agentsy/tokens for budget management, @agentsy/memory for persistence
   - Capabilities: Prompt compression, chunking, agentic patterns, safety/guardrails
   - Status: Implementation plan created (`plan/PROMPTS_TODO.md`), scheduled for Phase P13
