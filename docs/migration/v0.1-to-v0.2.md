# Migration from v0.1 to v0.2

This guide covers notable changes when upgrading the `agentsy` packages and extensions between version 0.1 and 0.2.

### Key Changes

- Package structure reorganized for better modularity and separation of concerns
- Consolidated stream processing and adapter utilities into core packages
- Updated retry utilities for better consistency across adapter implementations
- Centralized message transformation utilities for improved adapter reliability

### Migration Steps

This version focuses on internal refactoring with limited external-facing API changes. Most consumers should not need significant migration steps.

1. Review your imports and ensure they reference the correct package paths:

```typescript
// Verify imports are pointing to the correct packages
// Most existing imports should continue to work without changes
```

2. If you encounter any import issues, consult the updated package documentation.

3. Report any unexpected breaking changes via GitHub issues so we can improve migration guides.

### Notes

- This is primarily an internal refactoring with minimal consumer impact
- Future versions will expand adapter integration capabilities
- Focus on stability and API consistency for extension integrations

- This migration improves code reuse, reliability, and prepares the codebase for future extension adapter unification.

Please consult the updated API docs and release notes for detailed guidance.
