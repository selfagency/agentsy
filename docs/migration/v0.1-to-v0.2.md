# Migration from v0.1 to v0.2

This guide covers notable changes when upgrading the `agentsy` packages and extensions between version 0.1 and 0.2.

## Key Changes

- Introduced centralized retry utility in `@agentsy/retry` package.
- Updated all package usages to import retry utilities from `@agentsy/retry` for consistent retry behavior.
- Centralized message transformation for Mistral adapter into `@agentsy/adapters` using `toMistralMessages` for stable tool call ID mapping and validation.

## Migration Steps

1. Update retry imports in your extensions and providers:

```ts

```

- import { createRetryUtility } from '@agentsy/vscode/retry';

```ts
+ import { withRetry, retryWithBackoff } from '@agentsy/retry';
```

Note: `retryWithBackoff` is an alias for `withRetry`. Both exports are available for API compatibility.

2. Import and use `toMistralMessages` from `@agentsy/adapters` where converting chat messages for Mistral interaction.

3. Verify no VS Code-specific APIs leak into shared adapter utilities.

4. Read updated documentation and examples for usage.

## Notes

- The `mistral-models-vscode` package will adopt these new utilities in a subsequent planned update.

- This migration improves code reuse, reliability, and prepares the codebase for future extension adapter unification.

Please consult the updated API docs and release notes for detailed guidance.
