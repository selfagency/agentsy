# Migration from v0.1 to v0.2

**v0.2 represents major internal consolidation while maintaining API compatibility for existing consumers.**

## Overview

**v0.1** featured a distributed architecture with 30+ independent npm packages. **v0.2** consolidates these into a coherent, maintainable structure with 6 anchor packages and strategic subpath exports.

This is primarily a **structural refactoring** with **minimal consumer impact**:

- ✅ All existing APIs remain unchanged
- ✅ Published packages (`@agentsy/vscode`, `@agentsy/core`, `@agentsy/providers`, `@agentsy/orchestrator`) fully backward compatible
- ⚠️ Internal package paths updated via consolidation

## Phase C: Package Consolidation

### Phase C-1: Core Stream Processing (✅ Complete)

10 stream processing packages consolidated into `@agentsy/core` with subpath exports:

| Old Package           | New Path                   | Notes                                    |
| --------------------- | -------------------------- | ---------------------------------------- |
| `@agentsy/processor`  | `@agentsy/core/processor`  | LLMStreamProcessor, pipeline helpers     |
| `@agentsy/thinking`   | `@agentsy/core/thinking`   | Thinking tag extraction                  |
| `@agentsy/xml-filter` | `@agentsy/core/xml-filter` | XML/privacy scrubbing                    |
| `@agentsy/tool-calls` | `@agentsy/core/tool-calls` | XML/native tool call extraction          |
| `@agentsy/structured` | `@agentsy/core/structured` | JSON parsing/validation/repair           |
| `@agentsy/context`    | `@agentsy/core/context`    | Context block splitting/dedupe           |
| `@agentsy/formatting` | `@agentsy/core/formatting` | Formatting utilities (includes markdown) |
| `@agentsy/recovery`   | `@agentsy/core/recovery`   | Stream recovery strategies               |
| `@agentsy/retry`      | `@agentsy/core/retry`      | Generic retry logic                      |
| `@agentsy/sse`        | `@agentsy/core/sse`        | Server-Sent Events helpers               |

### Phase C-2: Provider Integration (✅ Complete)

Provider packages reorganized into `@agentsy/providers` anchor:

| Old Package                 | New Path                              |
| --------------------------- | ------------------------------------- |
| `@agentsy/normalizers`      | `@agentsy/providers/normalizers`      |
| `@agentsy/adapters`         | `@agentsy/providers/adapters`         |

### Phase C-3 & C-4: Orchestration (✅ Complete)

Orchestration packages consolidated into `@agentsy/orchestrator` anchor:

| Old Package      | New Path                                 |
| ---------------- | ---------------------------------------- |
| `@agentsy/agent` | `@agentsy/orchestrator/agent`            |
| `ag-ui` protocol | `@agentsy/orchestrator/ag-ui` (internal) |

## Migration Steps

### Step 1: Update Import Statements

Use search-and-replace in your IDE to update all imports:

**Mapping guide:**

```text
@agentsy/processor          → @agentsy/core/processor
@agentsy/thinking           → @agentsy/core/thinking
@agentsy/xml-filter         → @agentsy/core/xml-filter
@agentsy/tool-calls         → @agentsy/core/tool-calls
@agentsy/structured         → @agentsy/core/structured
@agentsy/context            → @agentsy/core/context
@agentsy/formatting         → @agentsy/core/formatting
@agentsy/recovery           → @agentsy/core/recovery
@agentsy/retry              → @agentsy/core/retry
@agentsy/sse                → @agentsy/core/sse
@agentsy/normalizers        → @agentsy/providers/normalizers
@agentsy/adapters           → @agentsy/providers/adapters
@agentsy/universal-client   → @agentsy/providers/universal-client
@agentsy/agent              → @agentsy/orchestrator/agent
```

**Example:**

```ts
// Before
import { LLMStreamProcessor } from '@agentsy/processor';
import { normalizeOpenAIChatChunk } from '@agentsy/normalizers';
import { extractXmlToolCalls } from '@agentsy/tool-calls';

// After
import { LLMStreamProcessor } from '@agentsy/core/processor';
import { normalizeOpenAIChatChunk } from '@agentsy/providers/normalizers';
import { extractXmlToolCalls } from '@agentsy/core/tool-calls';
```

## Behavioral Changes

✅ **None** — consolidation is purely structural. All APIs and behaviors remain identical.

## Backward Compatibility

✅ **No breaking changes** — this is a reorganization, not a redesign. All functionality is preserved; only import paths change.

**If you use `@agentsy/vscode`:** No action required. The extension automatically uses the new consolidated paths internally.

**If you use other packages directly:** Update imports using the mapping table above.
